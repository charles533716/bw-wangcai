# 优惠活动与“体育首存”活动需求总览

## 1. 文档说明

本需求拆分为后台与客户端两份PRD，研发可按负责模块分别查阅：

- [总站后台：活动管理与首存活动PRD](./总站后台-活动管理与首存活动-PRD.md)
- [客户端：优惠活动与体育首存活动PRD](./客户端-优惠活动与体育首存活动-PRD.md)

适用角色：产品、后台研发、客户端研发、服务端研发、测试。

## 2. 需求目标

1. 运营可在总站后台新增、修改、启停和删除首存活动。
2. 客户端可按分类查看活动，并进入体育首存详情页参与活动。
3. 系统根据后台配置提供场馆、奖励档位、图片和规则内容。
4. 服务端完成资格校验、钱包扣款、彩金发放和重复参与控制。

## 3. 核心业务流程

1. 运营新增首存活动，完成配置后以“禁用”状态保存。
2. 联调和测试通过后，运营将活动切换为“启用”。
3. 客户端根据站点、设备、状态和展示时间加载活动列表。
4. 用户进入详情页，选择场馆和奖励档位，输入申请金额。
5. 客户端实时展示可得彩金和流水要求。
6. 服务端校验资格与余额，成功后完成扣款、入账、发奖和参与记录。

## 4. 已确认的产品口径

### 4.1 两条示例活动

“体育首存送68%最高2000元”和“体育首存送68%最高1000元”按两条独立活动处理，各自拥有独立配置和详情页。

### 4.2 客户端活动条件

客户端“活动条件”模块中：

- 活动时间：读取后台“活动时间”。
- 活动对象、申请方式、条件摘要及示例数值：客户端固定文案。

固定文案必须与后台实际奖励档位一致，发布前由运营和测试核对。

### 4.3 客户端活动规则

客户端“活动规则”不是固定文案，必须读取后台“移动端详情”字段并按富文本展示。

### 4.4 钱包余额

中心钱包余额来自真实钱包服务。禁止使用随机数或localStorage模拟生产余额。

### 4.5 流水计算

本期按活动规则截图执行：

`提款流水要求 = (申请本金 + 可得彩金) × 流水倍数 + 申请本金`

## 5. 后台与客户端数据映射

| 后台配置 | 客户端用途 |
| --- | --- |
| 活动标题 | 列表卡片标题、详情页标题 |
| 活动分类 | 客户端分类筛选 |
| 活动标签 | 活动卡片标签 |
| 排序 | 活动列表排序 |
| 状态 | 是否展示、是否允许参与 |
| 展示设备 | 控制WEB或APP/H5展示 |
| 展示时间 | 控制活动列表可见时间 |
| 活动时间 | 列表时间、详情活动时间、参与校验 |
| 站点与活动场馆 | 详情页场馆下拉 |
| 奖励档位 | 类型下拉、彩金和流水计算 |
| 活动列表图 | 客户端活动卡片图片 |
| APP/H5主图 | 移动端详情页主图 |
| 移动端详情 | 移动端“活动规则”完整内容 |

## 6. 分工边界

| 模块 | 主要职责 |
| --- | --- |
| 总站后台 | 活动配置、启停、删除、图片和富文本维护 |
| 客户端 | 活动列表、详情展示、输入校验、计算预览和交互反馈 |
| 服务端 | 活动数据、资格校验、钱包处理、发奖、流水和幂等控制 |
| 测试 | 配置与展示映射、金额计算、参与资格、资金一致性和异常流程 |
