# 总站后台：活动管理与首存活动PRD

## 1. 需求范围

本文件仅描述总站后台功能：

- 活动管理 > 活动列表。
- 选择活动类型。
- 新增首存活动。
- 修改首存活动。

## 2. 活动列表

### 2.1 查询条件

| 查询项 | 规则 |
| --- | --- |
| 活动名称 | 支持关键词查询和回车查询 |
| 活动类型 | 下拉单选，默认全部 |
| 站点 | 下拉单选，受管理员站点权限限制 |
| 查询日期 | 选择开始日期和结束日期 |
| 查询/重置 | 查询刷新列表；重置恢复默认条件 |

### 2.2 列表字段

依次展示：时间、站点、活动编码、活动名称、活动类型、开始时间、结束时间、活动排序、热门排序、状态、活动设置。

### 2.3 状态与操作

- 状态使用开关，状态文案显示在开关右侧。
- 开启显示“启用”，关闭显示“禁用”。
- 启用活动不可删除，“删除”按钮置灰。
- 禁用活动允许删除，删除前二次确认。
- 点击“设置”进入对应活动修改页。

### 2.4 新增活动

- 页面只保留“新增活动”入口。
- 点击后先打开“选择活动类型”弹窗。
- 可选类型顺序：新人礼、签到、首存活动、通用活动。
- 其他未完成类型置灰，不可点击。
- 选择“首存活动”后进入新增首存活动页面。

## 3. 新增/修改首存活动

### 3.1 页面差异

| 项目 | 新增 | 修改 |
| --- | --- | --- |
| 页面标题 | 新增首存活动 | 修改首存活动 |
| 活动标题 | 可填写 | 回显已有标题，可修改并保存 |
| 状态 | 默认禁用 | 回显当前状态，可修改 |
| 选择站点 | 支持多选和搜索 | 示例活动固定为旺财体育，不可修改 |

### 3.2 配置信息

| 字段 | 规则 |
| --- | --- |
| 活动标题 | 必填；新增和修改时均可编辑；用于客户端列表和详情标题 |
| 活动分类 | 必填、多选；体育、真人、电竞、彩票、棋牌、电子；不提供“全类别” |
| 活动标签 | 必填、单选；最新、日常、新人、VIP、限时、豪礼、返水、无标签 |
| 排序 | 必填、正整数；数值越小越靠前 |
| 状态 | 启用/禁用；新增默认禁用 |
| 展示设备 | 全部、Web、APP/H5；全选与子项双向联动 |
| 活动周期 | 指定时间、永久 |
| 展示时间 | 控制客户端可见时间；指定时间选起止日期，永久只选开始日期 |
| 活动时间 | 控制参与时间；指定时间选起止日期，永久只选开始日期 |

时间规则：

- 指定时间：开始日期和结束日期均必填，结束日期不得早于开始日期。
- 永久活动：客户端显示“长期活动”。
- 活动未开始或已结束时，服务端必须禁止参与。

### 3.3 申请模式

- 固定显示“每个账户仅一次”。
- 控件禁用，不允许修改。
- 重复参与限制最终由服务端校验。

### 3.4 站点与场馆

新增时可选站点：旺财体育、DW体育、财神体育。支持多选和搜索。

| 站点 | 可选场馆 |
| --- | --- |
| 旺财体育 | 旺财体育、熊猫体育、IM体育、旺财真人、旺财彩票、旺财棋牌、DB棋牌、博雅棋牌、高登棋牌、旺财电竞、DB电竞、IM电竞、旺财电子、DB电子、PG电子 |
| DW体育 | DW体育、DW真人、WM真人、DW彩票、TC彩票、DB电子、PG电子 |
| 财神体育 | 财神体育、熊猫体育、IM体育、财神真人、WM真人、DB电子、PG电子 |

场馆交互：

- 每个站点单独展示一组场馆，默认展开。
- 支持收起和展开。
- 勾选“全选”后选中全部场馆。
- 全部子项被选中时，“全选”自动选中。
- 每行最多展示10个子项，换行后与首个具体场馆对齐。

### 3.5 VIP等级

首存活动不展示“VIP等级要求”。业务上新用户参与时按VIP0处理。

### 3.6 赠送形式与奖励档位

赠送形式支持：比例金额、固定金额。

比例金额表格字段：序号、转账金额、赠送比例、赠送上限、流水要求、操作。

固定金额表格字段：序号、转账金额、赠送金额、流水要求、操作。

操作规则：

- 点击“添加”新增奖励档位。
- 至少保留1条档位。
- 档位支持删除，最后1条不可删除。
- 切换赠送形式时表格位置和整体高度保持稳定，不出现明显抖动。
- 表格宽度截至“操作”列，不展示多余空白表格区域。

字段规则：

- 转账金额：该档位最低申请金额。
- 赠送比例：比例赠送时使用。
- 赠送上限：比例赠送的封顶金额。
- 赠送金额：固定金额赠送时使用。
- 流水要求：本金与奖励对应的流水倍数。
- 数字字段不得为负数。

## 4. 页面配置

| 端 | 图片配置 | 数量 | 必填 | 格式 |
| --- | --- | --- | --- | --- |
| WEB | 活动列表图 | 1张 | 否 | jpg/jpeg/png，不超过2M |
| WEB | WEB端主图 | 1张 | 否 | jpg/jpeg/png，不超过2M |
| APP/H5 | 活动列表图 | 1张 | 否 | jpg/jpeg/png，不超过2M |
| APP/H5 | APP/H5主图 | 1张 | 否 | jpg/jpeg/png，不超过2M |

- 上传后显示缩略图。
- 支持删除后重新上传。
- 不配置WEB端背景图和H5分享图。

## 5. 内容详情

| 字段 | 用途 | 必填 |
| --- | --- | --- |
| WEB端详情 | WEB客户端活动规则内容 | 否 |
| 移动端详情 | APP/H5客户端“活动规则”内容 | 否 |

- 两个字段均使用中文富文本编辑器。
- 移动端详情保存的富文本内容由客户端原样读取并展示在“活动规则”区域。
- 正式上线的首存活动应配置移动端详情；为空时客户端不展示活动规则卡片。

## 6. 保存与校验

保存前校验：

1. 活动标题、活动分类、活动标签、展示设备必填。
2. 展示时间、活动时间符合日期规则。
3. 至少选择1个站点和1个具体场馆。
4. 至少配置1条有效奖励档位。
5. 启用活动前，运营应完成图片、移动端详情和客户端固定条件文案的核对。

保存成功后返回活动列表并刷新数据；保存失败时保留已填写内容并展示失败原因。

## 7. 提供给客户端的数据

首存活动详情至少包含：

- 活动ID、活动编码、活动标题、活动分类、活动标签、排序和状态。
- 展示设备、展示时间和活动时间。
- 站点、场馆及奖励档位。
- WEB/APP/H5活动列表图和主图。
- WEB端详情、移动端详情。

## 8. 后台验收标准

1. 活动列表查询、状态切换、设置和删除规则正确。
2. 新增活动必须先选择活动类型。
3. 新增首存活动默认禁用，修改页正确回显已有数据。
4. 修改活动标题后可正常保存，活动列表和客户端读取最新标题。
5. 活动分类支持多选且不存在“全类别”。
6. 不展示VIP等级要求。
7. 站点、场馆全选、收起和展开交互正确。
8. 奖励档位添加、删除和赠送形式切换正确。
9. 图片支持上传、预览和删除。
10. 移动端详情可正常保存并提供给客户端。
11. 启用活动不可删除，禁用活动可删除。
