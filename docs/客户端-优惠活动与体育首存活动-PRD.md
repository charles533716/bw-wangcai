# 客户端：优惠活动与“体育首存”活动PRD

## 1. 需求范围

本文件仅描述客户端功能：

- 优惠活动列表和分类筛选。
- 体育首存活动详情。
- 中心钱包余额刷新。
- 场馆、奖励档位和申请金额选择。
- 彩金与流水计算预览。
- 立即参与和结果反馈。

## 2. 数据来源

### 2.1 后台动态配置

客户端读取后台配置的内容：

| 后台数据 | 客户端用途 |
| --- | --- |
| 活动标题 | 列表卡片标题、详情页标题 |
| 活动分类 | 分类Tab筛选 |
| 活动标签 | 卡片右上角标签 |
| 排序 | 活动列表排序 |
| 状态 | 是否展示、是否允许参与 |
| 展示设备 | 控制WEB或APP/H5展示 |
| 展示时间 | 控制活动列表可见时间 |
| 活动时间 | 列表时间、详情活动时间、参与校验 |
| 活动场馆 | 详情页“选择场馆”下拉 |
| 奖励档位 | “选择类型”下拉、彩金和流水计算 |
| 活动列表图 | 活动卡片图片 |
| APP/H5主图 | 移动端详情页主图 |
| 移动端详情 | “活动规则”完整内容 |

### 2.2 客户端固定内容

“活动条件”模块中，只有“活动时间”读取后台。以下内容均为客户端固定文案：

- 活动对象：新会员。
- 申请方式说明。
- 申请金额、红利百分比、红利上限和流水倍数的条件摘要。

注意：“活动规则”不属于固定文案，必须读取后台“移动端详情”。

## 3. 两条示例活动

客户端列表展示两条独立活动：

1. 体育首存送68%最高2000元。
2. 体育首存送68%最高1000元。

两条活动分别使用各自的活动ID、标题、图片、时间、场馆、奖励档位和移动端详情。

## 4. 优惠活动列表

### 4.1 页面结构

- 左侧分类导航，右侧活动卡片列表。
- 底部保留客户端主导航。
- 默认选中“全部”。

分类顺序：全部、体育、真人、电竞、彩票、棋牌、电子。

VIP仅作为活动标签，不作为分类Tab。

### 4.2 分类筛选

- 点击分类后，只展示后台“活动分类”包含该分类的活动。
- 多分类活动可同时出现在多个分类Tab。
- “全部”展示当前用户可见的全部活动。

### 4.3 活动卡片

卡片展示：

- APP/H5活动列表图。
- 活动标题。
- 活动时间：长期活动或日期范围。
- 活动标签，如新人、限时；“无标签”时不展示角标。

卡片不展示额外活动描述，不在图片左下角增加“体育场馆专属”等冗余标签。

### 4.4 展示与排序

只展示同时满足以下条件的活动：

1. 状态为启用。
2. 当前站点符合活动站点范围。
3. APP/H5在展示设备范围内。
4. 当前时间处于展示时间内。
5. 活动未被删除。

按后台“排序”字段升序展示；排序相同时按创建时间倒序。

### 4.5 卡片交互

- 点击卡片，根据活动ID进入对应详情页。
- 返回列表时保留原分类和滚动位置。
- 活动已禁用、删除或不存在时，展示“活动不存在或已下架”。

## 5. 体育首存活动详情

### 5.1 页面结构

从上至下展示：

1. 返回按钮和活动标题。
2. APP/H5主图。
3. 活动条件。
4. 中心钱包余额。
5. 活动申请表单。
6. 温馨提示。
7. 活动规则。

### 5.2 顶部信息

- 标题读取后台活动标题。
- 主图读取后台“APP/H5主图”。
- 图片保持固定比例和圆角，不拉伸变形。
- 主图不叠加“体育场馆专属”等额外文字。

### 5.3 活动条件

动态内容：

- 活动时间读取后台配置。
- 永久活动显示“长期活动”。
- 指定时间显示“YYYY-MM-DD至YYYY-MM-DD”。

固定文案：

- 活动对象：新会员。
- 申请方式：请在此活动页面的下方选择参加活动的场馆，并确认中心钱包余额充足。输入金额点击立即参与即可（参与前，请不要将申请金额转入场馆钱包）。

2000元专题固定条件：

| 申请金额 | 红利百分比 | 红利上限 | 流水倍数 |
| --- | --- | --- | --- |
| ≥100元 | 68% | 2,000元 | 12 |

1000元专题固定条件：

| 申请金额 | 红利百分比 | 红利上限 | 流水倍数 |
| --- | --- | --- | --- |
| ≥100元 | 68% | 1,000元 | 12 |

固定条件与后台奖励档位不一致时，以服务端实际配置为参与依据，并禁止该版本上线。

### 5.4 中心钱包余额

- 余额读取真实钱包接口，使用千分位并保留2位小数。
- 点击刷新按钮后图标旋转，请求期间按钮禁用。
- 成功提示“余额刷新完成”，并更新客户端全局资产状态。
- 失败提示“余额刷新失败，请稍后重试”，保留原余额。
- 禁止使用随机数或localStorage作为真实余额来源。

### 5.5 选择场馆

- 下拉展示后台为当前活动配置的场馆。
- 只展示当前用户所属站点下的可用场馆。
- 默认选中第一项，用户可切换。
- 无可选场馆时禁止参与，提示“当前活动暂无可参与场馆”。

### 5.6 选择类型

- 选项来自当前活动后台奖励档位。
- 多条档位按后台顺序展示。
- 默认选中第一档。
- 切换档位后重新计算彩金和流水要求。

比例档位文案：`存{转账金额}元送{赠送比例}%，最高送{赠送上限}元，{流水倍数}倍流水`

固定档位文案：`存{转账金额}元送{赠送金额}元，{流水倍数}倍流水`

### 5.7 申请金额

- 仅允许输入非负数字，最多保留2位小数。
- 提示文案：“请输入转入场馆申请优惠的金额”。
- 显示当前档位起存额度。
- 申请金额不得低于起存额度。
- 申请金额不得超过中心钱包可用余额。

### 5.8 金额计算

设申请本金为 D，赠送比例为 R%，赠送上限为 L，固定赠送金额为 F，流水倍数为 M。

- 比例彩金：`B = min(D × R%，L)`。
- 固定彩金：`B = F`。
- 流水要求：`T = (D + B) × M + D`。

示例：申请100元，赠送68%，流水12倍，则彩金为68元，流水要求为：

`(100 + 68) × 12 + 100 = 2,116元`

输入为空或不合法时，可得彩金和流水要求均显示“¥ 0.00”。

### 5.9 立即参与

点击前，客户端校验：

1. 已选择场馆和奖励档位。
2. 已输入有效申请金额。
3. 金额达到起存额度。
4. 中心钱包余额充足。

点击后：

- 按钮进入加载状态，禁止重复点击。
- 服务端校验活动状态、活动时间、新用户资格、体育投注记录、重复参与、互斥活动、场馆、档位和余额。
- 成功后提示“参与成功”，刷新中心钱包和场馆钱包余额。
- 失败时恢复按钮，并展示服务端返回的明确原因。

资金扣减、场馆入账、彩金发放和参与记录必须由服务端原子化处理。客户端计算结果仅用于预览。

### 5.10 温馨提示

- 位于“立即参与”按钮下方，与按钮左对齐。
- 使用小字号灰色静态文案，不使用闪烁、呼吸或脉冲动画。
- 文案：温馨提示：此活动，仅可参与一次，不可重复参与。

### 5.11 活动规则

- 规则内容必须读取后台“移动端详情”字段。
- 按后台保存的富文本格式展示，包括编号、换行、加粗等样式。
- 客户端不得自行写死或重新拼装规则内容。
- “移动端详情”为空时，不展示活动规则卡片。
- 后台更新移动端详情后，客户端重新进入页面或刷新详情时展示最新内容。

## 6. 异常提示

| 场景 | 提示文案 |
| --- | --- |
| 活动不可用 | 活动不存在或已下架 |
| 活动未开始 | 活动尚未开始 |
| 活动已结束 | 活动已结束 |
| 未选择场馆 | 请选择场馆 |
| 未选择档位 | 请选择活动类型 |
| 未输入金额 | 请输入申请金额 |
| 金额过低 | 申请金额不能低于起存额度 |
| 余额不足 | 中心钱包余额不足 |
| 不符合资格 | 当前账号不符合活动参与条件 |
| 已参与 | 此活动仅可参与一次，不可重复参与 |
| 系统异常 | 系统繁忙，请稍后重试 |

## 7. 客户端验收标准

1. 分类Tab、筛选结果和活动排序正确。
2. 仅展示符合站点、设备、状态和展示时间的活动。
3. 卡片正确展示图片、标题、时间和标签。
4. 详情页正确读取标题、主图、活动时间、场馆和奖励档位。
5. 活动条件除活动时间外均显示固定文案。
6. 活动规则完整读取并展示后台“移动端详情”。
7. 中心钱包刷新使用真实余额并同步全局资产。
8. 彩金封顶、固定彩金和流水计算正确。
9. 金额不足、余额不足和重复参与均被正确拦截。
10. 重复点击不会产生重复扣款或重复发奖。
11. 温馨提示位置、样式和文案正确。
12. 常见移动端尺寸下无重叠、截断和横向滚动。
