# 站点安全设置设计说明

## 范围

在“总控后台 > 站点管理 > 站点列表 > 配置”页面中，于“站点综合配置”右侧新增“安全设置”Tab。现有配置 Tab 和业务逻辑不调整。

## 页面结构

- 页面标题：实名信息唯一绑定设置
- 顶部统一说明：开启后，同一实名信息在当前站点内只能归属一个用户。用户解绑、删除收款方式或注销账号后，历史归属关系仍保留，不允许其他用户绑定。
- 配置项：手机号唯一绑定、银行卡唯一绑定、支付宝唯一绑定、微信唯一绑定。
- 每个配置项使用开关控件，默认全部开启。
- 页面右上角提供“保存安全设置”按钮。

## 交互

- 用户可以分别开启或关闭四个配置项。
- 点击保存后，原型保留当前站点的开关状态并提示“安全设置保存成功”。
- 不调用真实后端接口；使用原型本地状态模拟保存行为。

## 验收标准

1. “安全设置”位于“站点综合配置”右侧。
2. 四个配置项名称、统一说明文案与需求一致。
3. 四个开关首次进入时均为开启状态。
4. 修改并保存后，再次进入该 Tab 时仍展示已保存状态。
5. 页面在常用桌面宽度下排列整齐，无文字或控件重叠。
